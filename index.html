Register

  • Experimenters primarily set up experiments through a website and access resources through SSH.
  • Experimenter creates an account on the WiNEST platform and an initial project.
  • Project serves as a container for profiles, experiments, users, and accounting.
  • Most important thing is to be able to verify affiliation with a research institution.
  • Should be possible to onboard existing CloudLab and GENI users.

Registered

  • A project owner account requires manual review by WiNEST staff.
  • A private admin page will streamline and simplify the review process.
  • The system will send an email to the experimenter once the account is approved or with the reason for rejection.
  • Project owners will be able to invite or approve other users to their project without review from WiNEST staff.

Login

  • User login will be secured by best practices including HTTPS and secure password storage.

Dashboard

  • Dashboard page shows resources created by the experimenter.
  • Main resources are experiment profiles and running experiments.
  • Nothing has been created yet, but the researcher can launch one of the publicly available experiment profiles or create a new one.

Dynamic Backhaul Profile

  • Profile creation asks for descriptive information as well as a way to supply, generate, and visualize the Rspec.
  • This experiment creates wireless links between certain towers using our dynamic backhaul capability and reserves computing resources at two sites.
  • We will create an interface that allows researchers to view and select from possible wireless paths using line of sight and distance constraints.
  • The experiment execution engine will enforce constraints (#links <= #radios) and resolve conflicts with other running experiments.

Sensor Profile

  • This experiment profile pulls in data streams from some sensors and uses edge computing to preprocess the data.
  • The network topology shows two kinds of computing resources: a VM provisioned in the NOC "cloud", and a container provisioned in close proximity to the sensors.
  • VMs can be created from a pool of supported disk images.
  • Container images can be pulled from a public Docker repository or built from from a git repository.

Sleep Scheduling Profile

  • This experiment profile reserves an Open Cellular base station and two connected client units (CPEs).
  • The clients have some edge computing capabilities through Paradrop, and this experiment launches a container on one of them for collecting measurements.
  • In this case, the client radio itself is not programmable, so attaching a container is necessary, e.g. if the experimenter wants to generate traffic.
